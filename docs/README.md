# Documentation

This directory contains all project documentation.

## Structure

- **reference/** - Quick reference guides (tech summary, architecture overview)
- **architecture/** - Detailed architecture documentation and diagrams
- **api/** - Complete API documentation (endpoints, schemas, examples)
- **operations/** - Operations runbooks, troubleshooting, and incident response
- **adr/** - Architecture Decision Records (technical decisions)

## Quick Guide

### For Developers
- Start with [reference/TECHNICAL-SUMMARY.md](./reference/TECHNICAL-SUMMARY.md)
- Review [architecture/](./architecture/) for system design
- Check [api/](./api/) for API endpoints

### For Operations
- See [operations/](./operations/) for runbooks and procedures
- Review monitoring setup in [operations/monitoring/](./operations/)

### For Architecture Decisions
- Check [adr/](./adr/) for documented decisions
- Create new ADRs when making significant technical choices

## Getting Started

1. Read the main [README.md](../README.md)
2. Review [TECHNICAL-SUMMARY.md](../TECHNICAL-SUMMARY.md)
3. Explore architecture docs in `architecture/`
4. Check API docs in `api/`

## Contributing

When adding documentation:
- Use clear, concise language
- Include examples where helpful
- Keep diagrams up to date
- Link related documents
